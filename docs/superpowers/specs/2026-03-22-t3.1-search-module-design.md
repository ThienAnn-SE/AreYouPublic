## Architecture

### Files

| File | Purpose |
|------|---------|
| `src/piea/modules/search.py` | `SearchClient` + `SearchModule` + data models |
| `config/data_brokers.json` | Broker domain blocklist (min 20 entries) |
| `tests/unit/test_search.py` | Unit tests with `respx` mocks (~90% coverage target) |

### Class responsibilities

**`SearchClient`**
- Wraps Google CSE HTTP API (single endpoint: `customsearch/v1`)
- One public method: `search(query: str) ‚Üí tuple[SearchHit, ...]`  (tuple for consistency with `SearchQueryResult.hits`)
- Owns: `httpx.AsyncClient`, API key, engine ID, timeout
- Raises: `ModuleAPIError` on 4xx/5xx, `ModuleTimeoutError` on timeout, `RateLimitExceededError` on 429

**`SearchModule(BaseModule)`**
- Owns: `SearchClient` instance, `frozenset[str]` of broker domains (loaded from `config/data_brokers.json`)
- Implements: `async def execute(self, inputs: ScanInputs) -> ModuleResult` (must be async to satisfy `BaseModule` contract)
- Builds up to 3 queries via dynamic priority (Option B)
- Deduplicates results across queries by URL
- Runs broker domain check on every result
- Produces `ModuleFinding` entries for search exposure and data broker hits

## Mandatory: Data Structure Design Checkpoint

> **This checkpoint is required before writing any implementation code.**
> The shapes below must be reviewed and confirmed before Phase 5 begins.

### `SearchHit` ‚Äî single API result (pure DTO)

```python
@dataclass(frozen=True, slots=True)
class SearchHit:
    title: str
    snippet: str
    url: str           # canonical link
    display_link: str  # registered domain shown by Google (e.g. "linkedin.com")
```

*Rationale:* frozen + slots ‚Üí safe to store in sets, zero memory overhead, no mutation risk.

### `SearchQueryResult` ‚Äî output of one query execution

```python
@dataclass(frozen=True, slots=True)
class SearchQueryResult:
    query: str
    hits: tuple[SearchHit, ...]   # tuple, not list ‚Äî frozen dataclass must be truly immutable
    quota_exhausted: bool = False
```

*Rationale:* `quota_exhausted` flag lets `SearchModule` short-circuit remaining queries without raising, enabling partial results to be returned. `tuple` is used instead of `list` because `frozen=True` only prevents field reassignment ‚Äî a `list` field remains mutable (`.append()` works). `tuple` enforces the stated immutability contract.

### Broker registry ‚Äî loaded at init

```json
// config/data_brokers.json ‚Äî extended format with per-broker opt-out URLs
{
  "brokers": [
    {"domain": "spokeo.com",       "optout_url": "https://www.spokeo.com/opt_out/new"},
    {"domain": "beenverified.com", "optout_url": "https://www.beenverified.com/app/optout/search"},
    ...
  ]
}
```

```python
# In SearchModule.__init__:
self._broker_domains: frozenset[str]           # O(1) lookup set
self._broker_optout: dict[str, str]            # domain ‚Üí opt-out URL
```

*Rationale:* `frozenset` ‚Üí O(1) membership test, immutable after construction. `_broker_optout` provides per-broker opt-out URLs for the `remediation_action` field in FR-5.2 findings. When a broker has no configured opt-out URL, fall back to `"https://{domain}"`.

> **Note:** `remediation_action` must use the broker's **opt-out URL** (e.g. `https://www.spokeo.com/opt_out/new`), NOT the profile page URL. The profile URL goes in `evidence["url"]`.

### Evidence dict typing (L003 compliance)

All intermediate evidence dicts must be typed `dict[str, Any]` (not `dict[str, object]`) before being passed to `ModuleFinding(evidence=...)`. Per L003, `dict[str, object]` breaks `.get()` callers under mypy. The `ModuleFinding.evidence` field is declared as `dict[str, object]` in `base.py` (not changed by this task) ‚Äî the implementor must construct evidence as `dict[str, Any]` and pass it in; mypy accepts this because `Any` is compatible with `object` at the assignment site.

### Deduplication key

`dict[str, SearchHit]` keyed on `SearchHit.url`, insertion-order preserving, first-seen wins when the same URL appears in multiple queries.

## Query Construction (Option B ‚Äî Dynamic Priority)

**Method:** `_build_queries(inputs: ScanInputs) ‚Üí list[str]`

**Priority order:**

| Priority | Query template | Condition |
|----------|---------------|-----------|
| 1 | `"{full_name}"` | `full_name` is not None |
| 2 | `"{full_name}" "@{email_domain}"` | `full_name` and `email` both not None |
| 3 | `"{username}"` | `username` not None AND `len(queries) < 3` (fewer than 3 queries built so far) |
| 4 | `"{full_name}" "{username}"` | both not None AND `len(queries) < 3` |

> **Clarification:** The `len(queries) < 3` condition guards the total number of queries built so far ‚Äî it is NOT a length check on the username string. Both username characters and full-name+username combos are added only when the running query list has room remaining before hitting the hard cap of 3.

**Hard cap:** `queries[:3]`

**Edge cases:**
- All inputs `None` ‚Üí `[]` ‚Üí `execute()` returns early with `success=True, findings=[]`
- Duplicate queries (e.g. username == full_name) ‚Üí deduplicated via `dict.fromkeys(queries)`
- Email with no `@` ‚Üí domain-split query silently skipped

### Finding 1 ‚Äî Search exposure

| Field | Value |
|-------|-------|
| `finding_type` | `"search_exposure"` |
| `category` | `"search"` |
| `severity` | `LOW` (1‚Äì3 results) / `MEDIUM` (4‚Äì10) / `HIGH` (>10) |
| `title` | `"Public search results found for {identifier}"` |
| `evidence` | `{"query_count": int, "result_count": int, "urls": list[str] (first 10), "queries_used": list[str]}` |

Emitted only when `result_count > 0`.

### Finding 2 ‚Äî Data broker hit (one per matched domain)

| Field | Value |
|-------|-------|
| `finding_type` | `"data_broker_exposure"` |
| `category` | `"search"` |
| `severity` | `HIGH` |
| `title` | `"Profile found on data broker: {display_link}"` |
| `evidence` | `{"url": str, "title": str, "snippet": str, "broker_domain": str}` |
| `remediation_action` | `"Request removal at {optout_url}"` ‚Äî uses broker's opt-out URL from `_broker_optout[domain]`, NOT the profile URL |

### Broker domain match logic

```python
def _is_broker(self, hit: SearchHit) -> bool:
    # Strip "www." prefix using removeprefix (Python 3.9+), extract registered domain
    # NOTE: do NOT use lstrip("www.") ‚Äî it strips any leading chars from the set
    # {'w','.',} rather than the literal prefix, corrupting domains that start with 'w'
    domain = hit.display_link.lower().removeprefix("www.")
    parts = domain.split(".")
    registered = ".".join(parts[-2:])
    return registered in self._broker_domains
```

## Error Handling

| Exception | Where caught | Behaviour |
|-----------|-------------|-----------|
| `RateLimitExceededError` | `SearchModule.execute()` | Append error string, set `quota_exhausted=True`, stop remaining queries, return `ModuleResult(success=False, findings=accumulated_findings_so_far)` ‚Äî findings from queries that completed before the 429 are preserved |
| `ModuleAPIError` | Per-query in `execute()` | Append to `errors`, continue next query |
| `ModuleTimeoutError` | Per-query in `execute()` | Append to `errors`, continue next query |
| `FileNotFoundError` | `SearchModule.__init__()` | Re-raise immediately (fail-fast ‚Äî misconfiguration) |
| `json.JSONDecodeError` | `SearchModule.__init__()` | Re-raise immediately (fail-fast ‚Äî misconfiguration) |

Partial results are valid: if query 1 succeeds and query 2 times out, findings from query 1 are returned alongside the timeout error.
