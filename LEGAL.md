## 1. Purpose of this document

This document defines the legal and ethical boundaries for the Public Information Exposure Analyzer (PIEA). It is the canonical source for:

- The **terms of use** governing who may use the tool and how
- The **consent attestation text** that operators must agree to before every scan
- The **report disclaimer** that appears on every generated report
- The **data handling policy** covering what data is collected, stored, and deleted
- The **acceptable use policy** prohibiting misuse

Any update to this document that changes the consent text (Section 3) **must** be accompanied by a version bump to `CURRENT_CONSENT_TEXT_VERSION` in `src/piea/core/consent.py`.

## 2. Terms of use

### 2.1 Permitted use

PIEA is designed exclusively for the following purposes:

1. **Self-assessment** — individuals analyzing their own public information exposure
2. **Authorized security testing** — penetration testers or security consultants with documented written authorization from the subject
3. **Security awareness training** — demonstrating real-world exposure risks in an educational setting with the informed consent of participants

### 2.2 Prohibited use

The following uses are strictly prohibited:

1. Scanning any individual without their knowledge and explicit consent
2. Using the tool for stalking, harassment, intimidation, or surveillance
3. Using results to discriminate against, blackmail, or extort any person
4. Selling, redistributing, or publicly disclosing scan results without the subject's consent
5. Using the tool to circumvent any platform's authentication or access controls
6. Automated bulk scanning of multiple individuals
7. Any use that violates applicable local, national, or international law

### 2.3 No warranty

PIEA IS PROVIDED "AS IS" WITHOUT WARRANTY OF ANY KIND, EXPRESS OR IMPLIED, INCLUDING BUT NOT LIMITED TO WARRANTIES OF MERCHANTABILITY, FITNESS FOR A PARTICULAR PURPOSE, AND NON-INFRINGEMENT. THE AUTHORS AND CONTRIBUTORS SHALL NOT BE LIABLE FOR ANY CLAIM, DAMAGES, OR OTHER LIABILITY ARISING FROM THE USE OF THIS TOOL.

### 2.4 Limitation of liability

The risk score is an algorithmic estimate based on publicly available data. It may not reflect the complete exposure profile of the subject. PIEA does not guarantee the accuracy, completeness, or timeliness of any information presented.

The operators and developers of PIEA accept no liability for:

- Inaccurate or incomplete findings
- Decisions made based on scan results
- Downstream consequences of disclosing scan results
- Temporary unavailability of third-party data sources
- Changes to third-party API behavior or availability

## 3. Consent attestation text (version 1.0)

> **IMPORTANT:** This is the exact text presented to operators before a scan. Any change to this section requires a bump to `CURRENT_CONSENT_TEXT_VERSION`.

The following attestation must be acknowledged by the operator before any scan executes:

> **Consent Attestation**
>
> By proceeding, I attest to the following:
>
> 1. **Authorization:** I am either (a) scanning my own personal information, or (b) I possess documented written authorization from the individual whose information will be scanned.
>
> 2. **Purpose:** I am using this tool for one of the following purposes: personal self-assessment, authorized security testing, or security awareness training.
>
> 3. **Understanding:** I understand that this tool will query publicly available sources and APIs to collect information associated with the provided identifiers (name, email, username). No private or authentication-protected data will be accessed.
>
> 4. **Responsibility:** I accept sole responsibility for ensuring that my use of this tool complies with all applicable laws and regulations in my jurisdiction, including but not limited to data protection laws (GDPR, PDPD, CCPA, and others).
>
> 5. **No misuse:** I will not use the results of this scan for harassment, stalking, discrimination, blackmail, extortion, or any other harmful purpose.
>
> 6. **Data handling:** I understand that my IP address, timestamp, and consent attestation will be recorded in an audit log for compliance purposes.

The operator must select one of the following attestation types:

- **Self-scan:** "I am scanning my own information."
- **Authorized third-party scan:** "I have documented written authorization from the subject of this scan."

## 4. Report disclaimer

The following disclaimer appears on every generated report (as required by NFR-L5):

> This report was generated by the Public Information Exposure Analyzer (PIEA). All information presented was collected exclusively from publicly available sources through official APIs and public records. No private data was accessed, no authentication was bypassed, and no platform terms of service were violated during this scan. This report is provided for informational and security awareness purposes only. It does not constitute legal advice. The risk score is an algorithmic estimate and may not reflect the complete exposure profile. The operator is responsible for ensuring proper authorization was obtained before conducting this scan.

## 5. Data handling policy

### 5.1 What data is collected

| Data | Source | Purpose | Storage |
|------|--------|---------|---------|
| Operator IP address | HTTP request | Audit trail, rate limiting | PostgreSQL (audit_logs, consent_records) |
| Operator name | Consent form | Audit trail | PostgreSQL (consent_records) |
| Consent attestation | Consent form | Legal compliance | PostgreSQL (consent_records) — immutable |
| Target name (hashed) | Scan input | Scan execution | PostgreSQL (scans) — SHA-256 hash only |
| Target email (hashed) | Scan input | Scan execution | PostgreSQL (scans) — SHA-256 hash only |
| Target username | Scan input | Scan execution | PostgreSQL (scans) |
| API responses | Third-party APIs | Finding generation | Redis cache (TTL-limited), not persisted long-term |
| Scan findings | Derived from API responses | Report generation | PostgreSQL (findings) |
| Identity graph | Derived from API responses | Report generation | PostgreSQL (graph_nodes, graph_edges) |

### 5.2 What data is NOT collected

- Raw email addresses (stored as SHA-256 hashes only)
- Raw names (stored as SHA-256 hashes only)
- Passwords (never transmitted to the server; password checks use k-anonymity)
- Authentication tokens or session cookies from any platform
- Any data requiring authentication to access

### 5.3 Data retention and deletion

- **API response cache:** Automatically purged by Redis TTL (24 hours for breach data, 1 hour for profile data)
- **Scan results:** Retained until the user requests deletion
- **Consent records:** Retained for the configured compliance period; not deletable through the application (admin-only access per FR-1.2)
- **Audit logs:** Retained for the configured compliance period; not deletable through the application

### 5.4 Right to erasure (GDPR Article 17)

Users may request deletion of their scan results. Upon receiving a valid deletion request:

1. The scan record and all associated findings, graph nodes, and graph edges will be permanently deleted (cascade delete as defined in the database schema)
2. Consent records and audit logs are **excluded** from erasure as they serve a legitimate compliance purpose (GDPR Article 17(3)(b) — legal obligation)
3. Deletion will be completed within 72 hours of the request (NFR-L4)

## 6. Acceptable use policy

### 6.1 Rate limiting

To prevent misuse and ensure fair access:

- Maximum 10 scans per IP address per hour
- Respectful rate limits applied to all third-party API calls
- Operators who circumvent rate limits may be blocked

### 6.2 Platform compliance

PIEA accesses only:

- Official, documented public APIs (GitHub REST API, Mastodon API, etc.)
- Publicly accessible HTTP endpoints (username existence check via status code)
- Public DNS and WHOIS records

PIEA does **not**:

- Scrape web pages or use browser automation
- Bypass CAPTCHAs or anti-bot protections
- Access private or authentication-gated API endpoints
- Violate any platform's Terms of Service

### 6.3 Reporting misuse

If you believe this tool is being used in violation of these terms, or if your information has been scanned without your consent, please contact the system administrator immediately.
